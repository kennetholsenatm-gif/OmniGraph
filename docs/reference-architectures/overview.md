# Reference Architectures

This section contains **example deployment patterns**. These documents are not the
only supported way to run OmniGraph.

## How to use these examples

- Treat them as starting points
- Replace all placeholder values with your environment details
- Adapt identity, network, and provider choices to local standards

## Placeholder conventions

- `https://git.example.com/<org>/<repo>`
- `https://id.example.com`
- `<HOST_OR_IP>`
- `<OPERATOR_EMAIL>`
- `<CI_PLATFORM>`
